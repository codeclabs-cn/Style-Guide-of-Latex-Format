\section{Local Variables}\label{sec:local-variables}
Place a function's variables in the narrowest scope possible, and initialize variables in the declaration.

C++ allows you to declare variables anywhere in a function. We encourage you to declare them in as local a scope as possible, and as close to the first use as possible. This makes it easier for the reader to find the declaration and see what type the variable is and what it was initialized to. In particular, initialization should be used instead of declaration and assignment, e.g.,:
% \vspace{-\baselineskip}
\begin{minted}[mathescape,
    bgcolor=code_bg_con,
    linenos,
    numbersep=5pt,
    autogobble, % 左对齐
    breaklines,
    frame=lines,
    framesep=2mm]{C++}
int i;
i = f();      // Bad -- initialization separate from declaration.
\end{minted}
% \vspace{-\baselineskip}
\begin{minted}{C++}
int j = g();  // Good -- declaration has initialization.
\end{minted}
% \vspace{-\baselineskip}
\begin{minted}[mathescape,
    bgcolor=code_bg_con,
    linenos,
    numbersep=5pt,
    autogobble, % 左对齐
    breaklines,
    frame=lines,
    framesep=2mm]{C++}
std::vector<int> v;
v.push_back(1);  // Prefer initializing using brace initialization.
v.push_back(2);
\end{minted}
% \vspace{-\baselineskip}
\begin{minted}{C++}
std::vector<int> v = {1, 2};  // Good -- v starts initialized.
\end{minted}
Variables needed for \mintinline[breakanywhere,bgcolor=code_bg_pro]{C++}{if}, \mintinline[breakanywhere,bgcolor=code_bg_pro]{C++}{while} and \mintinline[breakanywhere,bgcolor=code_bg_pro]{C++}{for} statements should normally be declared within those statements, so that such variables are confined to those scopes. E.g.:
% \vspace{-\baselineskip}
\begin{minted}{C++}
while (const char* p = strchr(str, '/')) str = p + 1;
\end{minted}

There is one caveat: if the variable is an object, its constructor is invoked every time it enters scope and is created, and its destructor is invoked every time it goes out of scope.
% \vspace{-\baselineskip}
\begin{minted}[mathescape,
    bgcolor=code_bg_con,
    linenos,
    numbersep=5pt,
    autogobble, % 左对齐
    breaklines,
    frame=lines,
    framesep=2mm]{C++}
// Inefficient implementation:
for (int i = 0; i < 1000000; ++i) {
  Foo f;  // My ctor and dtor get called 1000000 times each.
  f.DoSomething(i);
}
\end{minted}
It may be more efficient to declare such a variable used in a loop outside that loop:
% \vspace{-\baselineskip}
\begin{minted}{C++}
Foo f;  // My ctor and dtor get called once each.
for (int i = 0; i < 1000000; ++i) {
  f.DoSomething(i);
}
\end{minted}
